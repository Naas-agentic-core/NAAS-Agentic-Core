doc(diagnostics): add diagnostic_report.md for dual memory collision

This commit adds a detailed diagnostic report mapping out the exact
paths, behaviors, and collision points between the LangGraph Checkpointer
memory model and the explicit history array used by Manual Agents (like
OrchestratorAgent).

It details how the shared use of `_build_graph_messages` creates a critical
state destruction scenario where manual agents lose conversation history
because it was stripped out under the assumption a LangGraph checkpointer
would handle it.

No production code was modified in this diagnostic exercise as per instructions.
